Ray Tracing

My current work in progress, October 22, 2018

This is a project programmed in C++ for my computer graphics course taught by Wojciech Jarosz. It revolves around creating a photorealistic scene with multiple light sources as well as a variety of spheres with various texture properties. These varied textures demonstrate different photoreactive tendencies of materials. i.e how ideal mirrored surfaces reflect light vs. how ideal matte surfaces scatter light vs. how semi-transparent or translucent surfaces refract light.

Linear Blend Skinng

A computer graphics project for CS77, October 16, 2018

This project, written in JavaScript leveraging WebGl, showcased different skinning techniques. It started off with a simple demonstration of rigid skinning then it progressed to a more complex vertex weight calculation applied to a singular joint for the purpose of linear blend skinning. Finally, I moved to designing a more complicated mesh built out around a skeleton consisting of 12 bones. Linear blend skinning was applied to this mesh to generate the image pictured above. UI controls enable users to move each joint independently of its parent's but each joint's motion propagates to its child nodes to create life like motion. This is achieved by means of a recursive hierarchy traversal function. This function applies the matrix transformation of each joint's local matrix to its successive children before converting them into the world space and rendering them.

Catmull-Clark Subdivision

Developed the recursive division functionality for "smoothed" vertex calculation.October 8, 2018

This is a project programmed in JavaScript guided by Wojciech Jarosz. Given an array of vertices and faces that comprise the input mesh (consisting of n-gons i.e could be any combination of triangles, quadrilaterals, both, etc.) The Catmull-Clark subdivision algorithm is applied a new subdivided mesh is returned. A hash-map is used to make sure that a vertex is not inserted more than once. The UI of this project allows the "depth" of the subdivision to be specified so that users can dictate linear, cubic or other such subdivisions to occur. The Catmull-Clark subdivision functionality was abstracted into three parts to minimize bugs and maximize readability as well as unit testing capabilities. First, a linear subdivision is applied (first unit test used here to make sure that the shape of the mesh doesn't actually change but each face is confirmed to have subdivided). Next, averaging is implemented so that vertices with four neighboring faces are "smoothed". Finally, a correction step is applied which essentially translates the smoothed vertex along the path between its "smoothed" location and its previous location by an interpolated factor η.
